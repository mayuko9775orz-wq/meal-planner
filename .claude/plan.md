# 献立プランナー v3 — 新アプリ設計プラン

## 概要
既存の `kondate-app-v2.html` をベースに、`料理/` ディレクトリに新しい献立プランナーアプリを構築する。
シングルHTMLファイル（SPA）として、iCloud同期を活かしどのデバイスからでも使えるようにする。

## 既存アプリの分析
v2は以下の機能を持つ単一HTMLファイル（約2200行）：
- **7タブ構成**: 検索 / マイレシピ / 基本レシピ / あかりレシピ / 献立 / 調理 / 買物
- **レシピDB**: 基本レシピ20件＋長谷川あかりレシピ22件（ハードコード）
- **並行調理タイムライン**: ガントチャート＋ステップリスト表示、リソース（コンロ1, コンロ2, オーブン, レンジ, 炊飯器, 手作業）ベースのスケジューリング
- **買い物リスト**: Notion連携あり
- **データ保存**: localStorage

## 新アプリの構成

### ファイル構成（単一HTML）
```
料理/
  index.html          ← メインアプリ（全コード内包）
```

### 要件と改善点

#### 1. 週間献立の自動生成（新機能）
- 月〜日の7日分の献立を一括自動生成
- 主菜・副菜・汁物のバランスを考慮
- 同じ食材・カテゴリが連続しないようにする
- 曜日ごとに手動で入れ替え・ロックも可能

#### 2. レシピ管理・検索（v2から改善）
- v2のレシピDBをそのまま引き継ぎ（基本レシピ＋あかりレシピ）
- マイレシピの作成・編集・削除
- キーワード、カテゴリ、タグでの検索・フィルタ
- 外部サイト取得機能はCORS制限で実用的でないため、手動登録に集約

#### 3. 買い物リスト生成（v2から改善）
- 献立の全材料を自動集約
- 同一材料はまとめて表示
- チェック機能付き
- Notion連携は維持

#### 4. 栄養バランスチェック（新機能）
- 各レシピにざっくり栄養カテゴリを付与（タンパク質/野菜/炭水化物/その他）
- 1日・1週間の栄養バランスをレーダーチャート風に可視化
- バランスが偏っている場合にアラート表示

#### 5. 並行調理タイムライン（v2から改善）
- v2のガントチャート＋ステップリスト表示を維持
- リソースベース（コンロ1/2, オーブン, レンジ, 炊飯器, 手作業）のスケジューリング
- **改善**: 依存関係を考慮したスケジューリング（同レシピ内のステップは順序保持）
- **改善**: 待ち時間（煮込み中など）を検出して他のタスクを挿入
- 調理タイマーとの連動

### UI構成（タブ）
1. **📅 週間献立** — メイン画面。7日分の献立ボード
2. **📖 レシピ** — レシピ一覧・検索・登録（基本/あかり/マイレシピ統合）
3. **⏱️ 調理** — 選択した日の献立から並行調理タイムライン生成
4. **🛒 買物** — 買い物リスト
5. **📊 栄養** — 栄養バランスチェック

### 技術仕様
- **フレームワーク**: なし（Vanilla JS）
- **CSS**: カスタムプロパティ活用、v2のデザインテーマ踏襲
- **データ保存**: localStorage（iCloud同期のためファイルベースも検討→localStorageで十分）
- **チャート**: Canvas API で自前描画（外部ライブラリ不要）
- **レスポンシブ**: モバイルファースト

### 実装順序
1. HTMLスケルトン＋CSS（タブUI、カード、レスポンシブ）
2. レシピDB（v2から移植）＋レシピ管理機能
3. 週間献立ボード＋自動生成ロジック
4. 並行調理タイムライン（v2ベース＋改善）
5. 買い物リスト（v2ベース）
6. 栄養バランスチェック
7. Notion連携
