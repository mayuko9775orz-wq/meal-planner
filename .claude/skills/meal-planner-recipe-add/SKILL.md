# レシピ追加ガイド

## DB選定フロー

```
レシピの性質は?
├─ 包丁不要・冷凍活用・15分以内・2-3工程 → limitRecipeDB
├─ 手順9以上・所要90分以上・本格 → challengeRecipeDB
├─ ホットクック使用 → hotcookRecipeDB
├─ 電気圧力鍋使用 → pressureRecipeDB
├─ カテゴリ「主食」(ごはん・麺・丼) → shushokuRecipeDB
└─ その他(主菜・副菜・汁物) → basicRecipeDB
```

IDは `getAllRecipes()` で自動生成: `b_タイトル`(basic), `h_タイトル`(hotcook), `l_タイトル`(limit), `c_タイトル`(challenge), `p_タイトル`(pressure)

## データ構造(1レシピ)

```js
{
  title: 'レシピ名',
  category: '主菜' | '副菜' | '汁物' | 'サラダ' | '主食' | 'ごはん・麺',
  desc: '短い説明(1文)',
  ingredients: ['材料名/分量', ...],  // スラッシュ区切り、例: 'キャベツ/1/4個'
  tags: ['タグ1', 'タグ2'],
  flavors: ['味噌', '醤油', ...],  // 味付け分類
  cookTime: 30,
  steps: [
    {text:'手順文', duration:5, resource:'リソース名', passive:false}
  ],
  nutrition: {protein:3, vegetable:2, carb:1, other:0},  // 0〜4

  // 限界レシピのみ
  limitLevel: 'limit',
  limitTags: [...],

  // 挑戦レシピのみ
  limitLevel: 'challenge',
  challengeTags: [...]
}
```

## resource の選び方(並行調理スケジューラに影響)

| 手順の内容 | resource |
|---|---|
| 切る・混ぜる・盛り付け | `手作業` |
| 鍋・フライパン(1つ目) | `コンロ1` |
| 鍋・フライパン(2つ目) | `コンロ2` |
| オーブン・トースター | `オーブン` |
| 電子レンジ | `レンジ` |
| 炊飯器 | `炊飯器` |
| ホットクック(まかせる) | `ホットクック` |

### passive: true をつけるべきもの
- 煮込み・蒸し(火にかけるだけで他の作業ができる)
- オーブン加熱中
- 浸水・解凍・冷却
- ホットクック稼働中

passive=true のステップは「料理人フリー」扱いで、並行作業の余裕時間になります。

## 限界レシピの判定基準

以下のいずれかを満たすもの:
- まな板・包丁ゼロ(冷凍カット・缶詰・カット済み)
- 火を一つしか使わない(一鍋・一フライパン・レンジのみ)
- 放置系(ホットクック等に入れるだけ)
- 調味料3種以内
- 所要15分以内 × 手を使う時間5分以内

**limitTags は以下6種類に固定**(増やさない):
`まな板不要` `冷凍OK` `一鍋` `放置系` `レンジのみ` `調味料3種以内`

## 挑戦レシピの判定基準

本格派。以下のいずれか:
- 複数工程(ソース別仕込み、下茹で+本煮込み等)
- 所要90分以上
- 特別な食材・技法
- 家族向け/祝祭日向け

**challengeTags 例**: `本格` `時間必要` `複数工程` `特別な食材` `オーブン` `家族`

## nutrition の目安

0〜4の5段階で4分類:
- **protein**: タンパク質源(肉・魚・卵・豆腐・豆類)
- **vegetable**: 野菜・食物繊維
- **carb**: 炭水化物(ごはん・麺・パン・いも類)
- **other**: 乳製品・果物・その他

### 例
- カレー(肉・野菜・ご飯): `{protein:3, vegetable:3, carb:2, other:0}`
- サラダ: `{protein:0, vegetable:3, carb:0, other:0}`
- 味噌汁: `{protein:1, vegetable:2, carb:0, other:1}`

## 追加作業のチェックリスト

1. [ ] 適切なDBを選定
2. [ ] 配列末尾にレシピオブジェクトを追加(1レシピ1行推奨、既存と同じ圧縮スタイル)
3. [ ] `cookTime` と `steps` の `duration` 合計がだいたい一致(厳密でなくてOK)
4. [ ] `resource` の割当は並行調理が成立するか確認
5. [ ] `passive: true` は放置可能なステップに明示
6. [ ] `nutrition` は0〜4で4分類すべて埋める
7. [ ] pressureRecipeDB は `getAllRecipes` で `source:'basic'` として push されるので、他のDBから既に pressure 判定済み
8. [ ] 新DBを追加する場合は `getAllRecipes` にも push処理を追記

## よくあるタグ(参考)

- 食材: `豚肉` `鶏肉` `牛肉` `魚` `豆腐` `卵` `野菜`
- 調理法: `煮物` `炒め` `蒸し` `焼き` `揚げ物`
- 料理系統: `和食` `洋食` `中華` `イタリアン` `アジア`
- コンテキスト: `時短` `作り置き` `常備菜` `お弁当` `朝食` `家族`
- 特殊: `冷凍OK` `ホットクック` `もっとクック` `圧力鍋` `オーブン`

## アンチパターン

- ❌ 材料の分量を省略(買物リスト集約で正しく動かなくなる)
- ❌ `resource` を適当に(並行調理の時間短縮が崩れる)
- ❌ `passive` 未指定(稼働率計算が狂う)
- ❌ `limitTags` を自由定義(6種類に固定)
- ❌ 既存レシピとタイトル重複(ID重複エラー)

## 追加後の確認

コミット前に以下を確認:
- ブラウザ(ローカル `file://` でもOK)でレシピ一覧に表示される
- 詳細画面で材料・手順が正しく表示される
- 該当カテゴリのフィルタでヒットする
- 限界・挑戦なら対応バッジが出る
