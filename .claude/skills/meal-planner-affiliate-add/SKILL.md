# アフィリエイト商品追加ガイド

## 楽天アフィHTMLからの情報抽出

ユーザーが楽天アフィリエイトの「リンクを作成」から生成されるHTMLを貼ってきます。以下を抽出:

### 必要な情報
1. **リンクURL(トラッキング付き)**
   - パターン: `https://hb.afl.rakuten.co.jp/ichiba/{ID1}.{ID2}.{ID3}.{ID4}/?pc=https%3A%2F%2F...`
   - `?pc=` の後ろは商品ページのエンコード済URL
2. **商品名**
   - `<a>` タグ内のテキストから取得
3. **価格**(任意、タグ設計の参考に)

### 簡略化するURL
HTMLには `&link_type=picttext&ut=...` など余計なパラメータが付きます。**除去してOK**:
```
https://hb.afl.rakuten.co.jp/ichiba/{ID}/?pc={encoded_item_url}
```
この形式で十分動作します。

## curatedProducts への追加場所

`index.html` の `DEFAULT_AFFILIATE.curatedProducts` 配列に、**カテゴリごとにまとまった位置**へ追加:

```js
const DEFAULT_AFFILIATE = {
  ...
  curatedProducts: [
    // 🍳 調理家電
    {name:'...', tags:[...], url:'...'},
    {name:'...', tags:[...], url:'...'},

    // 📦 収納・保存
    {name:'...', tags:[...], url:'...'},

    // 📚 書籍
    {name:'...', tags:[...], url:'...'},

    // 🍹 食材・飲料
    {name:'...', tags:[...], url:'...'},
  ]
};
```

## タグ設計(重要)

商品が**どのコンテキストで表示されるか**を決めるのがタグ。

### タグと表示場所の対応

| タグ | 表示される場所 |
|---|---|
| `ホットクック` | ホットクックレシピ詳細、ホットクック資源使用レシピ |
| `挑戦` | 挑戦レシピ詳細、「今日は凝る日」モード ON時の今日タブ |
| `限界` | 限界レシピ詳細、「がんばらない日」モード ON時の今日タブ |
| `冷凍` | 冷凍タグ持ちレシピ詳細、限界モード |
| `オーブン` | オーブン資源使用レシピ詳細 |
| `圧力鍋` | 圧力鍋タグ持ちレシピ詳細 |
| `書籍` | 栄養タブ末尾(`renderCuratedProductCards(['書籍','発達障害','一般'])`) |
| `発達障害` `ADHD` | 栄養タブ末尾 |
| `一般` | 全コンテキストで候補になる(タグマッチ弱め) |

### タグ決め方のフロー

1. **まず商品カテゴリ**: 家電 / 保存容器 / 書籍 / 食材 / 飲料 / 嗜好品
2. **次に対応レシピレベル**: 限界向け / 挑戦向け / 両方
3. **機器固有タグ**: ホットクック / ヘルシオ / オーブン / 圧力鍋 など
4. **ターゲット層**: 一人暮らし / 家族 / 大人
5. **最後に `一般`**: 広く露出したいなら追加、したくないなら省く

### 例

- ホットクック1L: `ホットクック,一人暮らし,一般` → ホットクック関連レシピで露出、一般的な全レシピでも候補
- 金麦: `晩酌,挑戦,大人` → **一般も限界もつけない**。挑戦モード・挑戦レシピ限定(疲れた当事者に出ないように)
- 多動脳: `書籍,ADHD,発達障害` → 栄養タブ専用。レシピには出ない

## 倫理配慮(必ず確認)

### 発達障害当事者向けアプリとしてNGな組み合わせ

- 酒類 × `限界` タグ → 疲れた日の自己治療パターンに直結。**絶対禁止**
- 酒類 × `一般` タグ → 通常レシピで押し付け。**避ける**
- 酒類 × 栄養不足アラート → **絶対避ける**

### 嗜好品・要配慮商品の扱い

**酒類・サプリ・「発達障害向け」明示商品** は以下の原則:

1. 疲弊している状態のモード(`限界`)では露出しない
2. 「栄養」等ヘルスケア文脈では露出しない
3. 「挑戦モード」や「本格料理を楽しむ日」など**余裕がある文脈限定**で露出
4. タグ設計で明確に隔離

### OKなパターン(大人の嗜好品)
- 金麦 → `晩酌,挑戦,大人`(余裕がある日の文脈のみ)

### NGなパターン
- ❌ 酒 → `限界,一般`(疲れた日に出てしまう)
- ❌ ADHDサプリ → `ADHD,一般`(栄養不足時に誤誘導)

## コミット流れ

1. `DEFAULT_AFFILIATE.curatedProducts` 配列に追加
2. 「💎 厳選商品リンク機能を拡張」や「書籍◯冊追加」などの意味あるメッセージでコミット
3. `main` に merge & push でGitHub Pages反映(1〜2分)

## 楽天以外のアフィリエイト

### もしもアフィリエイト
サービス案件(nosh / Oisix / つくりおき.jp 等)は `noshUrl` `oisixUrl` `tsukuriokiUrl` の専用フィールド:

```js
const DEFAULT_AFFILIATE = {
  ...
  noshUrl: 'https://af.moshimo.com/af/c/click?a_id=...',  // もしもアフィ
  oisixUrl: '...',
  tsukuriokiUrl: '...',
};
```

これらは`renderMealKitCard` が参照し、文脈別に表示:
- `limit` コンテキスト: nosh + tsukurioki(疲れた日の救世主)
- `empty` コンテキスト: nosh + oisix(献立が決まらない日)
- `shopping` コンテキスト: oisix + tsukurioki(買い物リスト下部)

### Amazonアソシエイト
`DEFAULT_AFFILIATE.amazonTag = 'xxxxx-22'` で設定。
材料検索リンクに自動付与される。

## 追加後の動作確認

設定後、以下で露出するはず:

| 商品のタグ | 期待される表示場所 |
|---|---|
| `ホットクック` | ホットクック肉じゃが等の詳細末尾 |
| `挑戦` | ビリヤニ等の詳細末尾 & 凝る日モード |
| `限界` | 冷凍豚汁等の詳細末尾 & がんばらない日モード |
| `書籍` | 栄養タブ末尾 |

## よく使うテンプレ行

```
商品名(ブランド等) | 主軸タグ,特徴タグ,ターゲット,一般 | 楽天URL
```

例:
```
シャープ ホットクック KN-HW10G 1L | ホットクック,一人暮らし,一般 | https://hb.afl.rakuten.co.jp/...
```

## アンチパターン

- ❌ 全商品に `一般` をつける → 全レシピで出て押し付けがましい
- ❌ 倫理的配慮なしに酒・サプリ露出
- ❌ タグを自由定義しすぎる → どこで表示されるか分からなくなる
- ❌ 重複IDの商品追加(URL被ったら楽天側で別トラッキングされる)
